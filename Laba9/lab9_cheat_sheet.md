# ЛР9 Django: подсказка по библиотекам, командам, методам и логике

## 1) Что делает программа

Сайт реализует учет пробега автотранспорта и расхода топлива.

Основные сущности:
- сотрудники;
- водители;
- автомобили;
- документы «Путевой лист».

Основные задачи:
- хранить связи между водителями и автомобилями;
- создавать путевые листы;
- автоматически считать пробег и расход топлива;
- строить диаграммы по водителям и автомобилям с линией среднего значения.

## 2) Основные библиотеки и зачем они нужны

### Django (ядро проекта)
- django.db.models
  - описание таблиц базы данных (модели).
- django.views.generic
  - типовые классы представлений (например, CreateView).
- django.shortcuts
  - удобные функции рендера шаблонов и поиска объектов.
- django.db.models (Sum, Avg)
  - агрегаты для сумм и средних значений.
- django.db.models.functions (Coalesce)
  - подмена NULL на 0 при агрегации.
- django.core.exceptions.ValidationError
  - собственные проверки и сообщения об ошибках.
- django.http.JsonResponse
  - возврат JSON для динамического обновления формы.

### Стандартная библиотека Python
- decimal.Decimal
  - точные вычисления для километража и топлива без ошибок округления float.
- datetime.timedelta
  - удобно формировать интервалы времени (например, в тестах и seed-данных).

### Frontend
- Chart.js (через CDN)
  - рисует столбики и горизонтальную линию среднего на странице диаграмм.

## 5) Ключевые модели и методы

Файл: [Laba9/transport/models.py](Laba9/transport/models.py)

### Employee
- Хранит ФИО сотрудника.
- Используется как базовый справочник людей.

### Car
- Поля: марка, госномер, год выпуска, норма расхода (л/км).
- Норма расхода участвует в вычислении топлива в путевом листе.

### Driver
- Связь 1:1 с Employee.
- Связь many-to-many с Car.
- Позволяет закреплять несколько машин за водителем и наоборот.

### TripSheet
- Поля ввода: водитель, автомобиль, время выезда/заезда, начальный/конечный километраж.
- Поля вычисления: mileage (пробег), fuel_consumption (расход).

Главные методы TripSheet:
- clean()
  - проверяет корректность данных до сохранения:
    - время заезда позже времени выезда;
    - конечный километраж не меньше начального;
    - автомобиль закреплен за выбранным водителем.
- save()
  - вызывает full_clean();
  - вычисляет пробег;
  - вычисляет расход топлива;
  - сохраняет объект.

Формулы:

$$
Пробег = Конечный\_км - Начальный\_км
$$

$$
Расход\_топлива = Пробег \cdot Норма\_расхода\,(л/км)
$$

## 6) Логика работы программы по шагам

1. Пользователь открывает страницу создания путевого листа.
2. Выбирает водителя.
3. Форма запрашивает через API только те автомобили, которые закреплены за этим водителем.
4. Пользователь вводит время выезда/заезда и километраж.
5. При сохранении модель запускает проверки clean().
6. Если проверки пройдены:
   - рассчитывается пробег;
   - рассчитывается расход топлива.
7. Документ сохраняется в базу.
8. Списки водителей и автомобилей автоматически показывают актуальные суммы:
   - водители: суммарный пробег;
   - автомобили: суммарный расход топлива.
9. Страница диаграмм пересчитывает данные из базы и показывает:
   - синие столбики по объектам;
   - красную горизонтальную линию среднего значения.

Итог: при любом изменении документов численные итоги и графики обновляются автоматически, потому что строятся на текущих данных БД.

## 7) Что важно объяснить на защите

- Почему есть отдельные справочники (сотрудники, водители, автомобили): это нормализует данные.
- Почему использована связь many-to-many между водителями и авто: по заданию требуется двусторонняя множественность.
- Почему вычисления вынесены в модель TripSheet.save():
  - единая точка расчета;
  - одинаковое поведение для формы, админки и любых других источников записи.
- Почему есть clean():
  - защищает от логически неверных данных.
- Как формируются диаграммы:
  - агрегаты Sum/Avg на уровне базы;
  - визуализация через Chart.js.

## 8) Быстрый сценарий демонстрации преподавателю

1. Выполнить seed-данные: py -3.11 manage.py seed_transport_data.
2. Показать список водителей и подтвердить суммарный пробег.
3. Показать список автомобилей и подтвердить суммарный расход.
4. Открыть форму путевого листа:
   - выбрать водителя;
   - показать, что список авто фильтруется.
5. Создать новый путевой лист.
6. Показать обновившиеся таблицы.
7. Открыть диаграммы и показать изменение столбиков и средней линии.
