# Шпаргалка по лабораторной работе (Система обмена данными: Источник, Сервер, Визуализатор)

## Основные библиотеки

### 1. Django (`django`)
- Это основа веб-приложения, которая обеспечивает маршрутизацию, обработку запросов и рендеринг HTML-страниц.
- **`from django.shortcuts import render`**: отрисовывает вашу главную и единственную страницу (`index.html`) при заходе на сайт.
- **`from django.http import JsonResponse`**: отвечает за возврат ответов в формате JSON. Поскольку ваша страница общается с сервером через JavaScript (`fetch`), сервер должен отдавать данные и статусы работы не в виде полного HTML, а в удобном формате словарей (JSON).

### 2. pywin32 (`win32com.client`, `pythoncom`) (Важно!)
- Библиотека, предоставляющая Python доступ к Windows API и OLE/COM (Component Object Model) объектам. Именно она позволяет Python-коду "нажимать на кнопки" в Excel программно.
- **`win32com.client.Dispatch('Excel.Application')`**: главная команда, которая инициализирует приложение Excel и возвращает объект управления им.
- **`excel.Visible = True`**: делает интерфейс Excel видимым для пользователя (изначально фоновые процессы OLE скрыты).
- **`pythoncom.CoInitialize()`**: обязательная команда при работе с COM-объектами в рамках многопоточных фреймворков (таких как Django). Она инициализирует COM-библиотеки для текущего потока, чтобы избежать ошибок с доступом к Excel.

### 3. Встроенные библиотеки Python
- **`json`**: используется повсеместно. `json.loads(request.body)` дешифрует данные, пришедшие из браузера, а `json.dump()` записывает структуру и данные из «Источника» в файл `export_data.json`.
- **`os`**: нужен для безопасной работы с путями (`os.path.join(settings.BASE_DIR, 'export_data.json')`).
- **`collections.defaultdict`**: используется на этапе "Сервера" для очень удобной группировки курсов. Если ключа (даты) еще нет, словарь сам создает пустой список, куда потом метод `.append()` добавляет курс.
- **`datetime`**: используется для парсинга строковых дат (например, `11.04.2024`) и их правильной сортировки, чтобы график строился хронологически верно.

## Ключевые важные моменты (Архитектура)

1. **Файловый обмен (`export_data.json`)**
   Ваш проект симулирует работу трех независимых систем:
   - **Источник** отправляет POST-запрос с JSON-телом, который Django просто сохраняет в локальный файл на диск.
   - **Сервер** при обработке *не* получает данные напрямую от Источника в памяти. Он читает тот самый внешний файл, проверяет структуру (флаг `transmit`) и усредняет значения, отдавая ответ клиенту.
   - **Визуализатор** отправляет POST-запрос уже с усредненными данными, и Django передает их через `pywin32` напрямую в Excel.

2. **Модель OLE/COM взаимодействия**
   При работе с Excel через OLE, вы по сути обращаетесь к внутренним объектам VBA Excel (кто писал макросы — синтаксис идентичен). 
   - `workbook.Worksheets(1)` — доступ к листам.
   - `sheet.Cells(row, col).Value` — доступ к конкретной ячейке (нумерация рядов и колонок начинается с **1**, не с 0).
   - `chart.ChartType = 65` — магия чисел. 65 означает константу `xlLineMarkers` (линейный график с точками-маркерами).

## Основные команды (в терминале)

**Установка зависимостей:**
```bash
pip install django pywin32
```
*(Если `pywin32` устанавливается глобально, после его установки часто рекомендуют запустить скрипт постинсталляции: `python -m pywin32_postinstall -install`, чтобы зарегистрировать системные DLL-ок в Windows).*

**Команды Django (запускать из корня проекта, где лежит manage.py):**
```bash
# Запуск отладочного сервера (теперь сайт будет доступен по адресу 127.0.0.1:8000)
python manage.py runserver

# Подготовка и применение миграций для базы данных
# (Даже если вы не используете БД напрямую, Django она нужна для встроенных компонентов: сессий, админки)
python manage.py makemigrations
python manage.py migrate
```
