# ЛР8 Django: шпаргалка по работе программы

## 1) Что делает проект

Это учебный Django-сайт для лабораторной работы №8 по теме печатных форм.

По заданию реализовано:
- отчет по всем сотрудникам с колонками:
  - № п/п;
  - Сотрудник;
  - Начислено;
- печатная форма отчета;
- подсветка строк по правилам:
  - выше верхнего порога;
  - ниже нижнего порога;
  - между порогами;
  - кратно или не кратно выбранному числу;
- добавление, просмотр, редактирование и удаление записей начислений.

В этой работе вместо справочника 1С используется обычная таблица Django, потому что в веб-проекте так проще показать структуру данных и логику отчета.

---

## 2) Главная идея логики

Логика программы строится так:

1. Пользователь добавляет запись начисления через форму или админку.
2. Данные сохраняются в таблицу `SalaryAccrual`.
3. Страница отчета читает все записи из базы.
4. Для каждой строки рассчитывается состояние:
   - сумма больше верхнего порога;
   - сумма меньше нижнего порога;
   - сумма между порогами.
5. Дополнительно проверяется кратность суммы выбранному числу.
6. Шаблон выводит таблицу и применяет CSS-классы для подсветки.
7. Отдельная печатная форма открывается без лишней навигации и готова к печати.

## 4) Модель данных

Файл: [payroll/models.py](payroll/models.py)

### `SalaryAccrual`
Это одна запись начисления зарплаты.

Поля:
- `employee` — имя сотрудника;
- `amount` — сумма начисления;
- `accrual_date` — дата добавления записи;
- `comment` — необязательный комментарий.

### Важные элементы модели

- `__str__()`
  - Возвращает понятное строковое представление объекта.
  - Нужно для админки и отладки.

- `Meta`
  - `verbose_name` и `verbose_name_plural` задают русские названия в админке.
  - `ordering` задает сортировку по умолчанию.

- `get_absolute_url()`
  - Возвращает ссылку на страницу записи после сохранения.
  - Удобно для перехода на карточку объекта.

## 7) Главные функции в `views.py`

### `_parse_decimal(value, default)`

Что делает:
- безопасно превращает строку из GET-параметра в `Decimal`;
- если значение пустое или неправильное, возвращает значение по умолчанию.

Зачем нужно:
- чтобы отчет не падал из-за неправильного ввода в URL.

### `_parse_int(value, default)`

Что делает:
- безопасно превращает строку в целое число;
- если значение некорректно, возвращает стандартное число.

### `_build_row(accrual, position, lower_limit, upper_limit, divisor)`

Это главная функция подготовки строки отчета.

Она:
- берет одну запись начисления;
- определяет ее номер в отчете;
- сравнивает сумму с порогами;
- проверяет кратность выбранному числу;
- возвращает готовый словарь с данными и CSS-классами.

Именно эта функция отвечает за подсветку строк.

## 8) Как работает подсветка

Подсветка идет на уровне отчета.

Правила:
- если сумма выше верхнего порога, строка получает класс `row-high`;
- если сумма ниже нижнего порога, строка получает класс `row-low`;
- если сумма между порогами, строка получает класс `row-middle`;
- если сумма кратна выбранному числу, показывается метка `multiple`;
- если не кратна, показывается `not-multiple`.

Где это видно:
- [templates/payroll/report.html](templates/payroll/report.html)
- [templates/payroll/report_print.html](templates/payroll/report_print.html)
- [static/css/style.css](static/css/style.css)

Важно:
- подсветка сделана минималистично;
- цвета не перегружают интерфейс;
- для обычного просмотра и для печати используются одинаковые данные.
