# Шпаргалка по основным библиотекам и командам (Лабораторные работы)

Эта информация поможет уверенно ответить на вопросы при защите работ, включающих фреймворк Django, интеграцию через COM (Excel), создание бота на Telegram и систему распределения ролей.

---

## 1. Основные используемые библиотеки

### **Django (`django`)**
Это основной веб-фреймворк, на котором базируются работы:
- **`django.shortcuts (render, redirect)`**: используются для отрисовки HTML-шаблонов (`render`) или перенаправления пользователя на другие страницы (`redirect`).
- **`django.http (JsonResponse, HttpResponse)`**: необходимы для возврата JSON-ответов (например, при передаче данных обмена) или простых текстовых ответов.
- **`django.contrib.auth`**: встроенная система аутентификации Django. Мы использовали её функции `authenticate`, `login`, `logout` для управления сессиями пользователей.
- **`django.contrib.auth.models (User, Group)`**: стандартные модели для создания пользователей и распределения их по группам (ролям: Директор, Секретарь, Заместитель, Гость).
- **`django.contrib.auth.decorators (login_required, user_passes_test)`**: логика защиты представлений (views). Позволяет пускать на страницу только авторизованных пользователей или пользователей с определенной ролью.

### **Работа с файлами и структурой (`json`, `os`, `datetime`)**
- **`json` (`json.dump`, `json.load`)**: встроенная библиотека Python для сериализации и десериализации. В лабе №3 мы считывали данные из `request.body`, превращали их в Python-словари и сохраняли в файл `export_data.json` для "обмена".
- **`os` / `os.path`**: для корректного построения путей к файлам независимо от операционной системы (например: `os.path.join(settings.BASE_DIR, 'export_data.json')`).
- **`datetime`**: для форматирования дат и работы с хронологией (перевод строковых дат в объекты даты и их сортировка для графика).

### **Интеграция с Excel (`pywin32`)**
- **`win32com.client`**: мощная библиотека для работы с OLE/COM объектами Windows. Позволяет из Python программно открывать другие Windows-программы.
- **`win32com.client.Dispatch('Excel.Application')`**: команда запуска самого приложения Excel.
- **Внимание**: так как Django работает в многопоточном режиме (когда принимает запросы), для `win32com` важно вызывать `pythoncom.CoInitialize()` перед работой с COM-объектами, чтобы инициализировать библиотеки в конкретном потоке выполнения.

### **Telegram и парсинг (`telebot` / `requests` / `xml.etree.ElementTree`)**
*Если на защите зайдет речь про валютного бота:*
- **`telebot` (`pyTelegramBotAPI`)**: библиотека для работы с Telegram API через лонг-поллинг (`bot.polling()`).
- **`requests`**: библиотека для выполнения HTTP/HTTPS запросов (используется для скачивания данных с сайта ЦБ РФ).
- **`xml.etree.ElementTree (ET)`**: встроенный парсер XML, потому что ЦБ РФ возвращает курсы валют в формате XML.

## 2. Главные концепции (Что нужно понимать!)

1. **Паттерн проектирования Django (MVT):**
   - **M (Model - Модель):** Описание структуры БД в виде Python-классов (файл `models.py`). Django ORM сама превращает классы в SQL таблицы.
   - **V (View - Представление):** Логика (файл `views.py`). Функция, которая принимает запрос (`request`), достает данные из модели или обрабатывает бизнес-логику, и передает результат в шаблон.
   - **T (Template - Шаблон):** HTML-файл с тегами Django (`{% for %}`, `{{ variable }}`), который формирует конечную картинку для браузера.

2. **Связь URL и View:**
   Все запросы проходят через `urls.py`. Django смотрит на адресную строку, находит совпадение по маршруту (например, `path('login/', views.user_login)`) и передает работу конкретной функции `user_login`.

3. **Обмен данными (Источник -> Сервер -> Визуализация):**
   - **Источник:** Генерирует или собирает данные и отправляет их форматом JSON (POST-запрос).
   - **Сервер:** Принимает данные, проверяет их, сохраняет или агрегирует (усреднение курса) и возвращает готовый к отображению массив.
   - **Визуализатор:** Берет обработанные данные и рисует из них график, передавая их в COM-объект Excel.

4. **Система распределения доступов (Role-Based Access Control):**
   - **Авторизация vs Аутентификация**: *Аутентификация* отвечает на вопрос "Ты кто?" (ввод логина/пароля), а *Авторизация* отвечает на вопрос "Что тебе можно делать?" (проверка принадлежности к группе).
   - Мы привязываем пользователя к группе (например, "Director"), а в коде `views.py` проверяем: `if user.groups.filter(name='Director').exists(): ...`

## 3. Основные терминальные команды (Консоль)

1. **Запуск виртуального окружения** (перед работой):
   ```bash
   venv\Scripts\activate
   ```
2. **Запуск сервера Django:**
   ```bash
   python manage.py runserver
   ```
   *Запускает локальный тестовый сервер (обычно на `http://127.0.0.1:8000/`), чтобы вы могли открывать сайт в браузере.*

3. **Создание миграций:**
   ```bash
   python manage.py makemigrations
   ```
   *Читает ваш файл `models.py` и создает файлы-инструкции для изменения базы данных (переводит Python-код моделей в SQL-формат).*

4. **Применение миграций:**
   ```bash
   python manage.py migrate
   ```
   *Непосредственно вносит изменения в базу данных (создает таблицы: встроенные джанговские и ваши собственные).*

5. **Создание суперпользователя (администратора):**
   ```bash
   python manage.py createsuperuser
   ```
   *Запросит логин, email и пароль. Этот аккаунт используется для входа в панель администратора `http://127.0.0.1:8000/admin/`.*

**Совет перед защитой:** Если спросят, почему используются именно эти инструменты (например, `JsonResponse`, а не обычный `render` в лабе с обменом), отвечайте, что для архитектуры "клиент-сервер" современного типа проще обмениваться "чистыми" данными в формате JSON, которые клиентский JavaScript получает асинхронно с помощью механизма Fetch API, не перезагружая страницу целиком.
