# Шпаргалка по лабораторной работе №6

## 1. Суть ЛР6 (вариант 1)

Цель: показать совместную разработку и элементы документирования на примере рабочего Django-сайта.

Что сделано в проекте:
- Реализован мини-сайт "Журнал совместной разработки" (CRUD для записей).
- Исходный код модулей снабжен подробными комментариями и docstring.
- Добавлены блоки "Журнал изменений" в ключевых Python-файлах.

## 2. Принципы документирования кода (по методичке)

1. Комментарии должны быть понятны другому разработчику.
2. Каждый модуль должен иметь заголовочный комментарий (docstring модуля).
3. Каждая функция/метод должны иметь комментарий назначения, аргументов и результата.
4. Сложные участки (ORM-агрегации, обработка форм, маршрутизация) сопровождаются пояснениями.
5. Глобальные переменные и константы должны быть подписаны.
6. При изменениях фиксировать:
	- код специалиста;
	- дату;
	- причину изменения;
	- какой фрагмент был изменен.

## 3. Архитектура проекта Django

Основные каталоги:
- `lab6site/` - конфигурация проекта (settings, urls, asgi, wsgi).
- `journal/` - предметное приложение (models, forms, views, urls, admin, tests).
- `templates/` - HTML-шаблоны.
- `static/` - CSS и другие статические файлы.

Связь слоев:
1. URL -> View.
2. View -> Model/Form.
3. View -> Template.
4. Template + CSS -> итоговый интерфейс в браузере.

## 4. Используемые библиотеки и модули

Фреймворк:
- `Django==5.2.1`

Ключевые модули Django:
- `django.db.models` - описание таблиц и полей БД.
- `django.forms` - формирование и валидация HTML-форм.
- `django.views.generic` - готовые классы ListView/DetailView/CreateView/UpdateView/DeleteView.
- `django.urls` - маршрутизация URL и reverse-ссылки.
- `django.contrib.messages` - сообщения пользователю после операций.
- `django.contrib.admin` - административная панель.

Стандартная библиотека Python:
- `os` и `sys` - запуск и окружение в `manage.py`.
- `pathlib.Path` - удобная работа с путями в `settings.py`.

## 5. Ключевые команды для работы

Установка зависимостей:

```bash
pip install -r requirements.txt
```

Миграции:

```bash
python manage.py makemigrations
python manage.py migrate
```

Создание администратора:

```bash
python manage.py createsuperuser
```

Запуск локального сервера:

```bash
python manage.py runserver
```

Запуск тестов:

```bash
python manage.py test
```

Проверка конфигурации проекта:

```bash
python manage.py check
```

1. Комментарии в каждом модуле приложения.
2. Docstring в классах и методах модели/форм/представлений.

## 7. Частые ошибки и быстрые решения

- Ошибка `NoReverseMatch`: проверить имя маршрута и `app_name` в `journal/urls.py`.
- Ошибка по статике: убедиться, что в шаблоне есть `{% load static %}`.
- Ошибка миграций: повторить `makemigrations`, затем `migrate`.
- Ошибка импорта Django: активировать виртуальное окружение и установить зависимости.
